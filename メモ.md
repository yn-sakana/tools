# 自分専用・自動化環境（オレオレRPA）構想書


## 1. コンセプト

「普段のPC作業から無駄なクリックと定型作業を完全に消し去る」ことを目的とした、完全自作の自動化環境。

会社PCの制約（管理者権限なし、外部ツールインストール不可）の中で、Windows標準機能のみで動作する軽量かつ拡張可能なシステムを構築する。


### 設計思想

1. **アドイン化とモジュール化:** 処理ごとにスクリプトファイルを分け、ポン置きで機能追加できる。
2. **設定の一元管理:** 環境依存データはすべて `config.json` に外出しし、コードを汚さない。
3. **起動したら全部使える、止めたら全部消える:** 基幹スクリプトが全機能を束ね、終了時に環境をクリーンアップ。レジストリ残骸なし。
4. **配布前提:** 他の人に渡せる形にする。「このスクリプトを走らせて」で導入完了。


---


## 2. 技術スタック

| 技術 | 役割 |
|---|---|
| **PowerShell** | 基幹スクリプト、OS操作、ファイル処理、CLI |
| **VBA** | Excel / Outlook / Word のOffice操作 |
| **Power Automate** | Teams / SharePoint のクラウド連携・イベント駆動 |
| **CSV / JSON** | VBA側とPowerShell側のデータ受け渡し |
| **md / txt** | 業務文書（メモ帳で編集、必要時Word変換） |

### 役割分担の原則

- **OS側の処理 → PowerShell**
- **Office内部の処理 → VBA**（Officeのネイティブ言語が一番安定する）
- **クラウド連携 → Power Automate**（認証まわりを自前で書かない）
- **VBAとPowerShellは直接繋がない。** 間にCSV/JSONファイルを置いてデータ連携する

### 3.2 基幹スクリプト（Main.ps1）の動作

```
起動
  ├─ config.json 読み込み
  ├─ 右クリックメニュー登録（レジストリ HKCU）
  ├─ フォルダ監視開始（Register-ObjectEvent）
  │
  │  動作中
  ├─ CLI対話ループ（コマンド入力待ち）
  │   ・ftree   → フォルダツリー表示
  │   ・search  → エクスポート済みデータ検索
  │   ・help    → コマンド一覧
  │   ・quit    → 終了
  │
  │  裏で動く
  ├─ フォルダ監視イベント（自動仕分け等）
  │
終了
  └─ 右クリックメニュー解除、監視停止、クリーンアップ
```


### 3.3 VBA↔PowerShell のデータ連携

VBA側がマスタデータを持ち、保存時にCSV/JSONを自動エクスポートする。
PowerShell側はエクスポート済みファイルを読むだけ。直接COMでOfficeを叩かない。

```
[Excel VBA]                        [PowerShell]
テーブル更新 → 保存時に自動出力  →  CSV/JSONを検索・表示
       ↑                              ↓
 マスタデータはExcelが正         検索・表示はPowerShellが得意
```

## 4. 実装予定の機能

### PowerShell側

- **右クリック → スクリプト実行:** 選択したファイルに対して、登録済みのスクリプトを実行
- **ファイル自動仕分け・リネーム:** config.jsonの規則に従い、監視フォルダのファイルを自動処理
- **フォルダツリー表示（ftree）:** 不要フォルダを省いた見やすい階層表示
- **エクスポート済みデータの CLI 検索:** キーを入力すると該当レコードを表示

### VBA側（Excel）

- **連絡先マスタ管理:** メールアドレス・保存先等を管理
- **操作ログ取得:** 登録したExcelの操作履歴を記録
- **テーブル → CSV/JSON 自動出力:** 保存時にエクスポート

### VBA側（Outlook）

- **メール振り分け・添付保存:** 連絡先別にフォルダ保存

### VBA側（Word）

- **テンプレートから文書生成**
- **md → Word 変換**（簡易）

### Power Automate

- **Teams / SharePoint 連携:** イベント駆動の自動化
